## 1. 이메일 템플릿 (`newsletter_template.html`)

받은편지함에서 안 깨지고 깔끔하게 보이도록 만든 HTML 이메일 템플릿입니다.

**설계 원칙**
- `<table>` 기반 레이아웃 → Outlook 등 구식 렌더링 엔진 호환
- 모든 스타일 인라인(`style="..."`) → Gmail의 외부 CSS 무시 대응
- 시스템 폰트(맑은 고딕 등) 사용 → 메일 클라이언트 웹폰트 차단 대응
- 프리헤더(받은편지함 미리보기 문구) · 다크모드 대응 포함
- 딥 네이비 + 골드 액센트 테마

**백엔드 연동 방법**
- `{{ }}` 로 표시된 자리표시자에 값을 채워 발송합니다.
- `<!-- [반복 시작] -->` ~ `<!-- [반복 끝] -->` 사이 블록을 뉴스 개수만큼 복사합니다.

| 자리표시자 | 설명 |
|-----------|------|
| `{{뉴스레터_이름}}` | 뉴스레터 제목 |
| `{{발송_날짜}}` | 발송일 |
| `{{프리헤더_문구}}` | 받은편지함 미리보기 문구 |
| `{{오늘의_핵심_요약}}` | 상단 한 줄 브리핑 |
| `{{카테고리}}` | 뉴스 분류 태그 |
| `{{뉴스_제목}}` | 뉴스 제목 |
| `{{요약_본문}}` | AI 요약 (3문장 이내) |
| `{{원문_링크}}` | 기사 원문 URL |
| `{{구독취소_링크}}` `{{설정_링크}}` | 푸터 링크 |

> ⚠️ 다크모드는 클라이언트마다 지원이 달라 완벽 보장은 어렵습니다. 라이트모드 기준으로 설계했습니다.

## 2. 보안 명세 (`보안명세_AI출력_방어.md`)

외부에서 긁어온 데이터와 AI 생성 텍스트를 안전하게 처리하는 규칙입니다.
두 가지 위협을 **서로 다른 지점**에서 막습니다.

- **HTML 이스케이프 (출력 단계)** — AI 텍스트의 `<`, `>` 등이 태그로 실행되는 것 방지
- **프롬프트 인젝션 방어 (입력 단계)** — 긁어온 텍스트 속 숨은 명령을 AI가 따르지 않도록 방어

## 3. 요약 검증 기준 (`요약_가독성_검증기준.md`)
