# AI 요약 가독성 검증 기준

> 담당: D (기획/데이터) — 요약 품질 검증
> 목적: AI가 만든 뉴스 요약이 "구독자에게 보내도 되는 수준"인지 판단하는 기준을 정한다.
> 사용법: 요약 하나당 아래 항목을 통과하면 발송, 하나라도 실패하면 재생성 또는 사람 검토.

---

## 1. 검증은 4가지 축으로 본다

| 축 | 핵심 질문 | 실패하면 |
|----|----------|---------|
| ① 정확성 | 원문 내용을 왜곡하지 않았나? | 신뢰도 붕괴 (제일 치명적) |
| ② 가독성 | 술술 읽히나? | 구독 취소로 이어짐 |
| ③ 형식 | 정해진 길이·구조를 지켰나? | 템플릿 깨짐 |
| ④ 중립성 | 특정 입장으로 치우치지 않았나? | 뉴스레터 신뢰 하락 |

---

## 2. 축별 세부 체크리스트

### ① 정확성 (가장 중요)
- [ ] 원문에 **없는 내용**을 지어내지 않았다 (환각 없음)
- [ ] 숫자·날짜·회사명·인명이 원문과 일치한다
- [ ] 원문의 핵심 논점을 빠뜨리지 않았다
- [ ] "~할 것이다"(추측)와 "~했다"(사실)를 뒤섞지 않았다

> 💡 정확성은 자동 판별이 어려움 → **샘플 검토**(전체의 10~20%를 사람이 원문과 대조)를 권장

### ② 가독성
- [ ] 한 문장이 너무 길지 않다 (권장: 한 문장 60자 이내)
- [ ] 전문용어에 짧은 설명이 붙어 있거나, 쉬운 말로 풀어썼다
- [ ] 주어·서술어가 명확하다 (번역투·비문 없음)
- [ ] 첫 문장만 읽어도 무슨 뉴스인지 감이 온다

### ③ 형식
- [ ] 요약이 **3문장 이내** (또는 팀이 정한 길이)
- [ ] 제목이 있고, 제목만으로 내용이 짐작된다
- [ ] 이모지·특수문자가 과하지 않다
- [ ] HTML 태그나 마크다운 기호(`**`, `#`)가 텍스트에 섞여 나오지 않았다

### ④ 중립성
- [ ] 광고·홍보 문구를 그대로 옮기지 않았다 ("최고의", "혁신적인" 등)
- [ ] 한쪽 주장만 담지 않았다 (특히 주식·정치 관련)
- [ ] 원문 출처가 편향된 경우, 단정적 표현을 피했다

---

## 3. 검증 프로세스 (실무 흐름)

```
AI 요약 생성
      │
      ▼
[자동 필터] ── 길이·금칙어·태그 검사 (통과 못 하면 재생성)
      │
      ▼
[샘플 검토] ── 전체의 10~20%를 사람이 원문과 대조
      │
      ▼
   발송 승인
```

- **자동 필터**로 거를 수 있는 것: 길이 초과, HTML 태그 잔존, 금칙어("최고", 광고 문구 등)
- **사람이 봐야 하는 것**: 환각(없는 내용), 미묘한 왜곡, 중립성 → 전수 검사는 불가능하니 샘플링

---

## 4. 재생성 vs 사람 검토 판단 기준

| 상황 | 조치 |
|------|------|
| 길이 초과, 태그 잔존, 형식 오류 | **자동 재생성** (프롬프트에 조건 강화) |
| 문장이 어색하거나 애매 | 재생성 1회 시도 후 안 되면 사람 |
| 숫자·사실이 원문과 다름 | **즉시 사람 검토** (자동 발송 금지) |
| 중립성 의심 (주식·정치) | **즉시 사람 검토** |

---

### 한 줄 결론
> 형식과 길이는 기계가 거르고, 사실과 편향은 사람이 표본으로 잡는다. 정확성은 속도보다 우선한다.
