# 테스트 중 기능 변경 정리 (발표 공유용)

이번 통합·검증 과정에서 **동작이 실제로 바뀐 것**만 모았습니다. 발표 때 "이렇게 보강했다"로 쓰시면 됩니다.
(문서·주석만 바뀐 건 제외. 템플릿 표현 정리도 제외 — 기능 변화가 아니라 서술 정리라서.)

## LLM 파트 (`LLM_fn.py`, `summarizer.py`)

### 환각(원문에 없는 내용을 지어냄) 막기 — 3중으로
1. **생성 프롬프트에 규칙 추가** — 요약 에이전트에게 "원문에 없는 수치(%·지수·금액 등)는 지어내지 말고, 확실한 숫자가 없으면 정성적으로만 써라", "한 이슈에는 묶은 기사들의 사실만 담고 다른 기사 내용을 섞지 마라(교차 오염 금지)".
2. **코드로 한 번 더 거름** — 요약에 원문에 없는 숫자가 섞이면 그 항목을 코드가 자동으로 빼냄(링크 무결성 검사와 같은 방식).
3. **검수 에이전트에 원문 전달** — 예전엔 링크만 넘겨 사실 검증을 못 했는데, 이제 원문 기사 텍스트를 함께 넘겨 근거 없는 사실·수치를 제거하게 함.

### 안정성
- 검수 에이전트가 JSON 객체가 아닌 값(배열 등)을 돌려줘도 죽지 않도록 방어.

## 백엔드 파트 (`api.py`, `subscriptions.py`, `db.py`, `pipeline.py`, `collectors/naver_news.py`, `notifiers/send_email.py`)

### 보안·인증
- **속도 제한 방식 변경** — 개인 대상 엔드포인트(본인 확인 코드 발송, `/subscribers/{email}` 조회·수정·삭제)를 IP가 아니라 **이메일 기준**으로 제한. 프론트가 백엔드를 서버끼리 호출해 IP가 다 같게 찍히는데, IP로만 세면 한 사람이 전체 한도를 먹거나 여러 IP로 확인 코드를 무차별 대입할 수 있어서. (+ X-Forwarded-For 인식, 경로 이메일 대소문자 정규화)
- **이메일 변경 시** 바꾸려는 주소가 이미 있으면(미확인 포함) 409로 막음 — 남이 가입 대기 중인 주소를 덮어써 없애지 않도록.
- 본인 확인 코드는 대소문자·앞뒤 공백 무시, 확인 토큰은 1회용 보장.

### 데이터·검증
- `GET /subscribers` 목록을 나눠 받기(`limit`/`offset`) + 전체 인원수를 `X-Total-Count` 헤더로(둘 다 검증 통과분 기준).
- 저장할 때 길이·개수 상한(이메일 254자·이름 100자·키워드 50개/각 50자), 발송 시각은 정수만(소수 거부).
- DB에 WAL 적용 — 동시 읽기/쓰기 안정.

### 발송 로직
- 일간 메일 제목을 구독자 키워드로 생성 — 예전엔 파이썬 리스트 표기(`['주식']`)가 제목에 그대로 새어 나갔음.
- `send_hour=24`(자정)를 다음 날 0시로 맞춤 — 발송 요일 판정도 다음 날 기준. (예전엔 24시가 영영 매칭되지 않았음)
- 주간 트렌드(별도 메일)를 보내는 앵커 요일도 같은 규칙으로 맞춤.
- 수신자별 발송 실패 격리 — 한 명의 SMTP 실패가 나머지 발송을 막지 않음.
- 뉴스 정제(`clean_text`)가 진짜 HTML 태그만 제거하도록 수정 — `<속보>` 같은 마커나 'A < B' 같은 부등호는 남김.
